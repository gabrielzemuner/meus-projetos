**Template Literals** são strings com crases (`` ` ``) que permitem:

- interpolar variáveis com `${...}`
- quebrar linha facilmente
- combinar expressões (inclusive `??`, `||`, ternário etc.)

---

## Exemplo: concatenação “antiga”

```js
const name = "Diego";
const message = "Bem-vindo, " + name;

console.log(message); // => "Bem-vindo, Diego"
```

---

## Exemplo: Template Literal (moderno)

```js
const name = "Diego";
const message = `Bem-vindo, ${name}`;

console.log(message); // => "Bem-vindo, Diego"
```

:::tip
Dentro do `${...}` você pode colocar **qualquer expressão** (não só variável).
:::

---

## Exemplo: usando `??` dentro do template

Aqui juntamos com Nullish Coalescing: se `name` for `null/undefined`, usa `"visitante"`.

```js
const name = null;
const message = `Bem-vindo, ${name ?? "visitante"}`;

console.log(message); // => "Bem-vindo, visitante"
```

---

## Resumo final

- `"texto " + var` → concatenação antiga
- `` `texto ${var}` `` → template literal
- `${...}` aceita expressão (ex.: `${name ?? "visitante"}`)

### 4) Fallback com `??` e `.trim()`

Dado:

```js
const name = "   ";
```

Crie uma mensagem que trate:
- se `name.trim()` ficar vazio, usar `"visitante"`
- senão, usar o nome

<details>
  <summary><strong>Ver resposta sugerida</strong></summary>

```js
const name = "   ";

const safeName = name.trim() ? name.trim() : null;
const msg = `Bem-vindo, ${safeName ?? "visitante"}`;

console.log(msg); // => "Bem-vindo, visitante"
```

</details>
